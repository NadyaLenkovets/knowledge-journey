# Prompt Lab — E2E тест-план (Playwright)

**Версия:** 1.0  
**Статус:** утверждён, реализован P0–P3  
**Проект:** `prompt-lab` (MVP, desktop-only, русский UI)

---

## 1. Цели

| Цель | Описание |
|------|----------|
| **Регрессия пользовательских сценариев** | Убедиться, что ученик может читать статьи, проходить inline-упражнения и многошаговые тесты без поломок UI |
| **Навигация и контент** | Все маршруты открываются, карточки ведут на правильные slug, ошибки 404-контента обрабатываются |
| **Критический flow теста** | Прогресс, «Проверить ответ» → feedback → «Далее →» → экран «Тест завершён» |
| **Дополнение Vitest** | Юнит-тесты покрывают `evaluate-*`; E2E — только интеграция в браузере |

**Не цель E2E:** проверка математики баллов, полнота контента статей, визуальный pixel-perfect, mobile/tablet.

---

## 2. Окружение

| Параметр | Значение |
|----------|----------|
| Браузер | Chromium (основной); опционально Firefox в CI |
| Viewport | `1280×720` (соответствует `min-width: 1280px`) |
| Locale | `ru-RU` |
| Base URL | `http://127.0.0.1:5173` (dev) / `http://127.0.0.1:4173` (preview) |
| Запуск сервера | `webServer` в `playwright.config.ts`: `npm run dev` или `npm run preview` после `build` |
| Скрипт | `npm run test:e2e` |

## 3. Приоритеты

### P0 — Smoke (обязательно в первой итерации)

Быстрые проверки (~2–3 мин прогона). Блокируют релиз при падении.

| ID | Сценарий | Ожидание |
|----|----------|----------|
| P0-01 | `GET /` | Редирект на `/main` |
| P0-02 | Главная `/main` | Hero «Практика промпт-инжиниринга», 3 карточки статей |
| P0-03 | Хедер | «Тесты» → `/tests`, логотип → `/main` (без пункта «Главная») |
| P0-04 | Страница тестов `/tests` | Заголовок «Тесты по темам», 3 карточки, пользовательский intro-текст (без FailureModePicker) |
| P0-05 | Несуществующая статья `/article/unknown` | «Статья не найдена», ссылка «На главную» |
| P0-06 | Несуществующий тест `/tests/unknown` | «Тест не найден», ссылка «К списку тестов» |

### P1 — Статьи и inline-упражнения

| ID | Сценарий | Ожидание |
|----|----------|----------|
| P1-01 | Карточка → статья LLM | URL `/article/kak-rabotayut-llm`, заголовок, обложка (SVG), badge «N мин чтения» |
| P1-02 | Inline LLM: singleChoice | Выбор правильного варианта → «Проверить ответ» → «Верно!» |
| P1-03 | Inline LLM: orderSteps | DnD/упорядочивание → проверка → feedback (корректный или с разбором) |
| P1-04 | Статья «Галлюцинации» | 2 упражнения: multipleChoice + spotTheHallucination — smoke (ответ + проверка) |
| P1-05 | Статья «Структура промпта» | matchPairs + failureModePicker — smoke |
| P1-06 | «← Все статьи» | Возврат на `/main` |

### P2 — Многошаговый тест (happy path)

Один полный прогон теста **`kak-rabotayut-llm`** (10 шагов, все 9 типов + доп.).

| ID | Шаг | Действие в E2E | Проверка |
|----|-----|----------------|----------|
| P2-01 | Старт | Карточка «Как работают LLM» → `/tests/kak-rabotayut-llm` | Заголовок, «Вопрос 1 из 10», прогресс 0→10% |
| P2-02 | 1–10 | Для каждого типа: минимально валидный ответ → «Проверить ответ» → «Далее →» | Индикатор шагов обновляется, процент растёт |
| P2-03 | Финал | После шага 10 | «Тест завершён», процент, «Баллы: …», разбор по вопросам |
| P2-04 | Навигация с итога | Ссылка к статье | `/article/kak-rabotayut-llm` |
| P2-05 | Назад | «← Все тесты» | `/tests` |

**Стратегия ответов:** хелпер `answerCurrentStep(page)` — по типу упражнения выбирает заведомо правильные варианты по тексту label из контента (стабильные строки в `exercises/index.ts`), не хардкод id в тестах.

### P3 — Негативные и UX (`e2e/specs/exercises-negative.spec.ts`)

| ID | Сценарий | Ожидание |
|----|----------|----------|
| P3-01 | «Проверить ответ» без выбора | Кнопка disabled или подсказка «Сначала выберите…» |
| P3-02 | Неверный singleChoice | «Неверно» + текст explanation.incorrect |
| P3-03 | Частичный multipleChoice | «Частично верно» (жёлтый статус) |
| P3-04 | Прогресс-индикатор | После ответа шаг N окрашен (зелёный/жёлтый/красный) |
| P3-05 | Второй тест smoke | Только шаги 1–2 для `galjucinacii` или `struktura-prompta` |

## 4. Покрытие типов упражнений (в P2)

| Тип | Шаг в `kak-rabotayut-llm` | Способ автоматизации |
|-----|---------------------------|----------------------|
| singleChoice | 1 | `getByRole('radio', { name: /…/ })` |
| trueFalse | 2 | `getByRole('radio', { name: /Верно|Неверно/ })` |
| multipleChoice | 3 | несколько `checkbox` по label |
| fillTheBlank | 4 | `getByRole('textbox')` / combobox |
| matchPairs | 5 | DnD: `dragTo` между зонами или клик-пары (если есть) |
| orderSteps | 6 | `dragTo` по `aria-label` «Перетащить шаг N» |
| promptBuilder | 7 | drag блоков в слоты |
| spotTheHallucination | 8 | клик по `aria-label` «Фрагмент N» |
| failureModePicker | 9 | `checkbox` по label рисков |
| trueFalse (доп.) | 10 | как шаг 2 |

Если DnD окажется нестабильным в CI — добавить **минимальные** `data-testid` только на drag-handle и drop-zone (не на весь UI).

## 5. Селекторы и устойчивость

**Предпочтение (в порядке приоритета):**

1. `getByRole` — кнопки «Проверить ответ», «Далее →», radio/checkbox с текстом option
2. `getByText` / `getByLabelText` — заголовки страниц, промпты (частичное совпадение)
3. `getByLabelText('Прогресс по вопросам')` — индикатор шагов
4. `data-testid` — **только** для DnD и при необходимости `exercise-shell` (по согласованию при реализации)

**Избегать:** CSS-классы Chakra (`css-*`), XPath, привязка к порядку DOM без роли.

## 7. CI и локальный запуск

```bash
npm install -D @playwright/test
npx playwright install chromium
npm run test:e2e          # headless
npm run test:e2e:ui       # опционально: UI mode
```

В `package.json`:

- `test:e2e` → `playwright test`
- `test:e2e:ui` → `playwright test --ui`

**Критерий успеха первой поставки:** P0 + P1 + P2 зелёные локально и в CI (если подключите).

## 8. Риски и митигация

| Риск | Митигация |
|------|-----------|
| DnD (@dnd-kit) нестабилен | `data-testid` на handles; увеличить `timeout`; fallback — keyboard где возможно |
| Долгий P2 (~10 шагов) | Один spec, параллель не нужна; шаги в `test.step()` |
| Анимация hero на `/main` | `await expect(...).toBeVisible()` без strict snapshot |
| Загрузка Google Fonts | `waitForLoadState('networkidle')` или отключить webfonts в CI |
| HMR vs preview | CI на `preview` после `build` — стабильнее dev |
